It is worth mentioning a few notes about commits since they can be forgotten or confused.

  1. A commit is similar to saving a Word document, closing Word, and shutting down your computer. Sure, you can go back and reverse your changes or keep editing, but it would take some time to start your computer back up and open the file. Same with commits. It is possible to reverse a commit, but it can be annoying to have to do so. Git gives you multiple stages in the finalizing process to help you catch any edits you forgot to make.

  2. If for whatever reason you find that you made a commit too early or did so accidentally, you can reset your current branch back to what it was before the commit, according to Git's knowledge. Details can be found in Git Procedure 13 in [cross-reference to target(s) "S_gittroubles" missing or not unique].

  3. It is possible to have multiple commits in one push. See Subsection  for more details.
